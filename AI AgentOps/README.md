## 비전 & 미션

### 🎯 비전

**"AI 에이전트가 모든 비즈니스 프로세스를 자동화하는 시대를 앞당긴다."**

파이고시프트는 2033년 $182B 규모로 성장할 AI 에이전트 시장의 선구자가 되어, 소상공인부터 대기업까지 모든 조직이 AI의 힘을 누릴 수 있는 플랫폼을 제공합니다.

### 🚀 미션

1. **진입장벽 낮추기:** 복잡한 기술 없이 누구나 AI 에이전트를 활용할 수 있는 환경 조성
2. **실제 가치 창출:** 단순한 자동화를 넘어 비즈니스 성과를 직접 증명하는 솔루션 제공
3. **의도적 혁신:** 기존의 틀을 깨고 새로운 가능성을 창조하는 기업 문화 구축

## 핵심 가치

### 1️⃣ **AgentOps 철학**

파이고시프트의 모든 제품은 **"에이전트가 주인공"**이라는 원칙 하에 설계됩니다. 인간은 에이전트를 지휘하고, 에이전트는 24/7 자율적으로 비즈니스를 운영합니다.

### 2️⃣ **선택과 집중**

바이브 코딩 2.0의 핵심인 **"무엇을 만들지 말아야 할지 아는 것"**을 실천합니다. 검증된 도구(Supabase, shadcn/ui, Stripe, Vercel)를 최대한 활용하여 핵심 기능에만 집중합니다.

### 3️⃣ **현업의 지혜**

낭만코딩(@romantic_coding)의 13단계 현업 프로세스를 에이전트 시스템에 이식하여, 단순한 기능 구현이 아닌 **"제품 성공"**을 추구합니다.

### 4️⃣ **투명한 의사결정**

모든 결정의 근거를 명확히 하고, 기술 부채를 주기적으로 정리하며, 실패로부터 배우는 조직 문화를 유지합니다.

## The 7 Parks 에이전트 군단

파이고시프트의 조직은 **"The 7 Parks"**라는 7명의 전문가 에이전트로 구성됩니다. 각 에이전트는 특정 분야의 최고 전문가로서, 마누스(Manus) 사령관의 지휘 아래 협력합니다.

### 🎖️ 조직도

```
                    마누스 (Manus)
                   총지휘 사령관
                        |
        ________________|________________
       |       |       |       |       |       |
    프로이트박  고든박  이노박  시박   프트박  트박
     (CEO)    (CSO)   (CTO)  (CMO)  (CFO)  (COO)
```

### 👥 각 에이전트별 역할

| 에이전트 | 역할 | 책임 | 주요 도구 |
| :--- | :--- | :--- | :--- |
| **프로이트 박** (CEO) | 전략 & 비전 | 사업 방향 설정, 가치 분석, 의사결정 | Gemini, Anti-gravity |
| **고든 박** (CSO) | 보안 & 준수 | 데이터 보호, 규정 준수, 리스크 관리 | OpenClaw, Zod |
| **이노박** (CTO) | 기술 & 혁신 | 아키텍처 설계, 기술 선택, 성능 최적화 | tRPC, Drizzle ORM |
| **시박** (CMO) | 마케팅 & 성장 | 시장 진입, 고객 확보, 브랜드 구축 | Gemini, PostHog |
| **프트박** (CFO) | 재무 & 자원 | 예산 관리, 자금 조달, ROI 분석 | 데이터 분석 |
| **트박** (COO) | 운영 & 실행 | 프로세스 최적화, 일정 관리, 품질 보증 | Sentry, Vercel |

### 🤖 마누스 (Manus) - 총지휘 사령관

마누스는 모든 에이전트를 조율하는 **메타 에이전트**입니다. 역할:

- **워크플로우 설계:** 13단계 프로세스에 따른 전체 실행 계획 수립
- **도구 간 데이터 흐름 제어:** Anti-gravity, OpenClaw, Gemini 간 최적의 협업 구조 설계
- **최종 검증:** 모든 결과물의 품질 보증 및 시장 적합성 확인
- **의사결정 지원:** 각 에이전트의 보고를 종합하여 경영진에게 최종 권고

## 6대 비즈니스 모델

### 🎯 선정 기준

**진입장벽이 낮고 AgentOps로 구현이 매우 용이한 분야**를 선정했습니다. 각 모델은 **V1 킬러 모델(소상공인 스마트 서기관)**을 시작으로 순차적으로 확장됩니다.

### 📊 6대 모델 상세

#### 1️⃣ **소상공인 스마트 서기관** (V1 킬러 모델)

**문제:** 소상공인은 마케팅, 회계, 고객 관리 등 여러 업무를 동시에 처리해야 하지만 전문 인력이 부족합니다.

**솔루션:** 
- 날씨, 행사, 계절 데이터를 분석하여 자동으로 SNS 홍보 콘텐츠 생성
- 매일 아침 최적의 시간에 자동 배포
- 고객 피드백을 학습하여 콘텐츠 개선

**기술 구현:**
```
입력 (가게 정보, 날씨, 이벤트)
    ↓
Gemini 2.0 API (콘텐츠 생성)
    ↓
자동 SNS 배포 (Instagram, Facebook, Naver)
    ↓
분석 & 최적화 (PostHog)
```

**예상 ROI:** 월 30만원 비용으로 연 500만원 매출 증대

**타겟:** 카페, 식당, 미용실, 소매점 (약 1,000만 개)

---

#### 2️⃣ **지능형 에너지 최적화**

**문제:** 중소 제조업체는 전력 비용이 운영 비용의 20~30%를 차지하지만, 최적화 방법을 모릅니다.

**솔루션:**
- 실시간 전력 데이터 모니터링
- 피크 시간대 자동 감지 및 부하 분산
- 주간 비용 절감 리포트 제공

**기술 구현:**
- IoT 센서 데이터 수집
- OpenClaw로 실시간 분석
- Gemini로 최적화 전략 제시

**예상 ROI:** 월 비용 절감 10~20%

---

#### 3️⃣ **제조 공정 안전 파수꾼**

**문제:** 산업 재해로 인한 인명 피해와 경제적 손실이 지속적으로 발생합니다.

**솔루션:**
- CCTV/센서 기반 작업자 안전 수칙 준수 실시간 감지
- 위험 행동 즉각 경고
- 주간 안전 리포트 및 개선 권고

**기술 구현:**
- 비전 AI (OpenClaw)로 작업자 행동 분석
- Gemini로 위험 수준 판단
- 자동 알림 및 기록

---

#### 4️⃣ **로컬 맞춤형 큐레이터**

**문제:** 여행자는 현지 정보 부족으로 최적의 경험을 놓칩니다.

**솔루션:**
- 실시간 교통, 맛집, 이벤트 데이터 통합
- 사용자 선호도 학습
- 1:1 맞춤 여행 가이드 제공

**기술 구현:**
- 지역 데이터 API 통합
- Gemini로 개인화된 추천
- PostHog로 사용자 행동 분석

---

#### 5️⃣ **행정 서류 자동화 에이전트**

**문제:** 공공 사업 신청서 작성은 복잡하고 시간이 많이 소요됩니다.

**솔루션:**
- 대화형 입력으로 필요한 정보 수집
- 공공 사업 신청서 자동 변환
- 규정 검토 및 오류 수정

**기술 구현:**
- Gemini로 대화형 정보 수집
- 템플릿 기반 자동 생성
- Zod로 검증

---

#### 6️⃣ **스타트업용 미니 보안 관제**

**문제:** 스타트업은 보안 위협에 노출되어 있지만 전담 인력이 없습니다.

**솔루션:**
- 클라우드 로그 자동 수집 및 분석
- 위협 탐지 및 즉각 차단
- 주간 보안 리포트 제공

**기술 구현:**
- OpenClaw로 로그 분석
- Gemini로 위협 판단
- Sentry로 모니터링

## 13단계 고도화 프로세스

파이고시프트는 낭만코딩의 **현업 13단계 프로세스**를 모든 비즈니스 모델에 적용합니다. 이는 단순한 기능 구현을 넘어 **"제품 성공"**을 보장하는 체계입니다.

### 📋 13단계 상세 설명

#### **Phase 1: 기획 (Planning)**

| 단계 | 이름 | 담당 에이전트 | 산출물 | 체크포인트 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | **PF** (Product Feasibility) | 프로이트 박 | 제품 가능성 분석 | 시장 기회 확인 |
| 2 | **FBS** (Feature Breakdown Structure) | 이노박 | 기능 분해 구조 | 기능 범위 명확화 |
| 3 | **RFD** (Requirement for Design) | 시박 | 설계 요구사항 | 디자인 방향 결정 |

**목표:** 무엇을 만들 것인가? 왜 만드는가? 누구를 위한가?

**마누스의 역할:** 각 단계의 산출물을 종합하여 전체 제품 비전 수립

---

#### **Phase 2: 리스크 체크 (Risk Check)**

| 단계 | 이름 | 담당 에이전트 | 산출물 | 체크포인트 |
| :--- | :--- | :--- | :--- | :--- |
| 4 | **FBD** (Feature Breakdown Design) | 이노박 | 기술 설계 | 기술 타당성 확인 |
| 5 | **RFE** (Requirement for Engineering) | 고든 박 | 엔지니어링 요구사항 | 보안 & 규정 검토 |

**목표:** 기술적, 보안적, 규정적 리스크 사전 차단

**마누스의 역할:** 리스크 우선순위 지정 및 대응 전략 수립

---

#### **Phase 3: 구현 & 검증 (Implementation & Verification)**

| 단계 | 이름 | 담당 에이전트 | 산출물 | 체크포인트 |
| :--- | :--- | :--- | :--- | :--- |
| 6 | **FUE** (Feature Update Engineering) | 이노박 | 코드 구현 | 기능 완성 |
| 7 | **RFQ** (Requirement for Quality) | 트박 | 품질 요구사항 | QA 계획 수립 |
| 8 | **FUQ** (Feature Update Quality) | 트박 | 테스트 & 검증 | 버그 제거 |

**목표:** 고품질의 제품 완성

**마누스의 역할:** 각 단계의 진행 상황 모니터링 및 병목 해결

---

#### **Phase 4: 런칭 결정 (Launch Decision)**

| 단계 | 이름 | 담당 에이전트 | 산출물 | 체크포인트 |
| :--- | :--- | :--- | :--- | :--- |
| 9 | **RFT** (Requirement for Test) | 트박 | 최종 테스트 계획 | 런칭 준비 완료 |
| 10 | **FUT** (Feature Update Test) | 트박 | 최종 테스트 실행 | 모든 테스트 통과 |
| 11 | **FL** (Feature Launch) | 프로이트 박 | 제품 출시 | 시장 진입 |

**목표:** 제품을 시장에 안전하게 출시

**마누스의 역할:** 최종 GO/NO-GO 결정

---

#### **Phase 5: 사후 회고 (Post-Mortem)**

| 단계 | 이름 | 담당 에이전트 | 산출물 | 체크포인트 |
| :--- | :--- | :--- | :--- | :--- |
| 12 | **FNL** (Feature New Learning) | 전체 | 학습 정리 | 개선 사항 도출 |
| 13 | **Post-Mortem** | 전체 | 회고 보고서 | 다음 버전 계획 |

**목표:** 성공/실패로부터 배우고 지속적 개선

**마누스의 역할:** 조직 전체의 학습 촉진 및 문화 정착

## 바이브 코딩 2.0: 18가지 규칙

파이고시프트는 상위 1% 빌더의 철학인 **"바이브 코딩 2.0"**의 18가지 규칙을 모든 개발에 적용합니다.

### 🛠️ 그룹 1: "만들지 말고 갖다 쓰세요"

#### Rule 1: 인증은 Supabase Auth를 쓰세요
- **왜?** 2주를 절약하고, 사용자는 로그인 화면에 감사하지 않습니다.
- **파이고시프트 적용:** Manus OAuth + Supabase Auth 조합으로 보안과 편의성 동시 확보

#### Rule 2: UI는 Tailwind + shadcn/ui 조합이 정답
- **왜?** Figma에서 실제 동작하는 화면까지 2~3시간이면 충분합니다.
- **파이고시프트 적용:** 모든 UI를 shadcn/ui로 구성하여 일관된 디자인 언어 유지

#### Rule 3: 결제는 Stripe 이외의 선택지가 없어요
- **왜?** PCI 컴플라이언스를 직접 감당하는 것은 위험합니다.
- **파이고시프트 적용:** V2에서 Stripe 통합 예정

---

### 🚀 그룹 2: "아키텍처보다 배포가 먼저"

#### Rule 4: API는 tRPC + Server Actions로 시작하세요
- **왜?** 사용자 0명인 제품에 REST API를 설계하는 것은 오버엔지니어링입니다.
- **파이고시프트 적용:** tRPC 11로 타입 안전성 확보, 나중에 필요시 마이그레이션

#### Rule 5: 상태 관리는 Zustand면 충분합니다
- **왜?** 12명이 쓰는 제품에 Redux를 넣는 것은 10만 명을 위한 아키텍처를 미리 짜는 것입니다.
- **파이고시프트 적용:** React Query + Server Components로 상태 관리 최소화

#### Rule 6: 배포는 Vercel 원클릭으로 자동화하세요
- **왜?** 서버 설정하고 SSH 키 관리하는 시간은 제품에 써야 할 시간을 갉아먹습니다.
- **파이고시프트 적용:** main에 push하면 자동 배포

---

### 🔐 그룹 3: "데이터는 지켜야 한다"

#### Rule 7: 데이터베이스는 Prisma + 매니지드 Postgres로 가세요
- **왜?** Raw SQL은 유지보수가 어렵고 보안 허점이 생기기 쉽습니다.
- **파이고시프트 적용:** Drizzle ORM + MySQL/TiDB로 타입 안전성 확보

#### Rule 8: 파일 업로드는 UploadThing이나 Cloudinary에 맡기세요
- **왜?** 스토리지, CDN, 파일 사이즈 제한까지 직접 구현하면 프로덕션에서 예상치 못한 방식으로 터집니다.
- **파이고시프트 적용:** S3 기반 파일 저장소 사용

#### Rule 9: API 키는 절대 코드에 하드코딩하지 마세요
- **왜?** GitHub은 퍼블릭 리포지토리를 자동 스캔합니다.
- **파이고시프트 적용:** 환경변수 관리 + .gitignore 필수

## 배포 가이드

- https://vercel.com 방문
- GitHub 계정으로 로그인

```bash
# Vercel CLI 설치
npm i -g vercel

# 프로젝트 배포
vercel
```

#### Step 3: 환경변수 설정
Vercel Dashboard에서:
1. Project Settings → Environment Variables
2. 다음 변수 추가:
   - `DATABASE_URL`: MySQL 연결 문자열
   - `GEMINI_API_KEY`: Google Gemini API 키
   - `VITE_APP_ID`: Manus OAuth App ID
   - `VITE_OAUTH_PORTAL_URL`: Manus OAuth Portal URL

#### Step 4: 배포
```bash
# main 브랜치에 push하면 자동 배포
git push origin main
```

```bash
# Sentry 계정 생성: https://sentry.io

SENTRY_DSN=https://your-dsn@sentry.io/project-id
```

```bash
# PostHog 계정 생성: https://posthog.com

VITE_POSTHOG_API_KEY=your-api-key
```
